# MotoCargo MVP Demo Script (Spanish-first)

## Goal
Show a realistic Seville restaurant procurement flow with contracts, compliance checks, and autonomous recurring order setup.

## Demo Accounts
- `restaurante-demo@motocargo.es` (restaurant role)
- `proveedor-demo@motocargo.es` (producer role, invite-only onboarding)

## Storyline (4 minutes)
1. Login as restaurante and show role-based landing.
2. Open home categories and enter `Carnicerías`.
3. Open provider catalog and add products to cart.
4. Go to checkout and show:
- Delivery window selection
- Vehicle profile selection
- Compliance badge and city-rule validation
5. Confirm order and show order confirmation timeline.
6. Navigate to contracts:
- Create draft
- Request one revision
- Accept contract (e-sign-lite audit trail)
7. Open recurring plan creation and show auto-confirm toggle.
8. Switch to producer account:
- Open today’s orders
- Progress order status to delivered
9. Open admin web:
- Show invitations
- Show city rules
- Show contracts and orders monitoring

## Closing Talking Points
- Solves Seville access restrictions and morning procurement pain.
- Contract becomes operational data, not just static legal text.
- Enables autonomous pipeline with recurring plans + compliance controls.
